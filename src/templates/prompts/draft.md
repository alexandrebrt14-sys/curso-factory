## Princípios de Andragogia (Malcolm Knowles)

Aplique rigorosamente os seis princípios da aprendizagem de adultos:

1. **Necessidade de saber**: abra cada módulo explicando POR QUE o aluno precisa dominar este tema — qual problema real ele resolve, qual oportunidade abre
2. **Autoconceito do aprendiz**: trate o aluno como profissional autônomo capaz de tomar decisões. Nunca seja condescendente. Use "considere", "analise", "avalie" em vez de "faça isso"
3. **Experiência prévia**: conecte novos conceitos com experiências que o aluno provavelmente já teve no trabalho. Use frases como "Se você já enfrentou...", "Na sua rotina profissional..."
4. **Prontidão para aprender**: demonstre aplicabilidade imediata. Cada conceito deve ter um cenário de uso real que o aluno pode aplicar hoje
5. **Orientação para problemas**: organize o conteúdo em torno de problemas reais, não de taxonomias abstratas. Comece com o problema, depois apresente a solução
6. **Motivação intrínseca**: conecte o aprendizado com crescimento profissional, autonomia e domínio. Evite motivadores externos superficiais

## Estrutura obrigatória do módulo

### 1. Abertura com Impacto (250–350 palavras)

- Comece com um dado surpreendente, um estudo de caso real ou uma pergunta provocativa (estilo HBR)
- Apresente o problema central que o módulo resolve, com dados concretos
- Conecte com o módulo anterior mostrando a progressão lógica (exceto módulo 1)
- Encerre com os **Objetivos de Aprendizagem** em formato de lista numerada, usando verbos de ação da Taxonomia de Bloom (analisar, avaliar, criar, aplicar — nunca "entender" ou "conhecer")

### 2. Fundamentação Conceitual

Desenvolva cada conceito com profundidade analítica:

- **Estrutura progressiva**: do fundamento teórico à aplicação prática
- **Evidências e dados**: cite pesquisas, estatísticas ou estudos de caso para cada afirmação relevante
- **Comparações estratégicas**: use tabelas comparativas para contrastar abordagens, ferramentas ou metodologias
- **Analogias sofisticadas**: conecte conceitos novos com domínios que o profissional já domina
- **Destaque de conceitos-chave**: use blocos de citação (>) para insights fundamentais
- **Alertas e armadilhas**: sinalize erros comuns com blocos de aviso

Formato obrigatório para conceitos-chave:

> **Conceito central:** [descrição concisa e memorável do conceito]

### 3. Análise de Caso ou Demonstração Prática

- Apresente um **estudo de caso real** (empresa, projeto ou cenário verificável) ou uma demonstração técnica detalhada
- Estruture com: Contexto → Desafio → Abordagem → Resultado → Lições Aprendidas
- Se o tema envolver código, comandos ou fórmulas, apresente em blocos de código bem comentados
- Inclua uma **tabela de decisão** ou **framework de análise** quando aplicável

### 4. Quadro Comparativo ou Síntese Visual

Inclua ao menos uma tabela estruturada por módulo:

| Critério | Opção A | Opção B | Opção C |
|----------|---------|---------|---------|
| ...      | ...     | ...     | ...     |

Ou um framework de decisão:

| Situação | Recomendação | Justificativa |
|----------|--------------|---------------|
| ...      | ...          | ...           |

### 5. Exercícios Práticos (mínimo 3, progressão de complexidade)

Para cada exercício:

- **Título descritivo** (não "Exercício 1")
- **Contexto profissional**: situe o exercício num cenário de trabalho real
- **Enunciado claro** com dados suficientes para resolução
- **Nível de complexidade**: Aplicação / Análise / Avaliação / Criação (Taxonomia de Bloom)
- **Critérios de avaliação**: o que define uma resposta excelente vs. adequada
- **Dica estratégica**: uma orientação que guie sem entregar a resposta

### 6. Síntese Executiva e Conexão (200–250 palavras)

- **Pontos-chave em lista**: recapitule as 4–6 ideias fundamentais do módulo
- **Checklist de aplicação**: liste 3–5 ações que o aluno pode executar imediatamente
- **Ponte para o próximo módulo**: mostre como o conhecimento adquirido será expandido ou aplicado
- **Referências recomendadas**: sugira 2–3 leituras/recursos complementares (artigos, livros, ferramentas)

## Diretrizes Editoriais (Estilo HSM/HBR/MIT Sloan)

### Tom e Linguagem
- Tom analítico e propositivo — nunca superficial ou genérico
- Linguagem direta, ativa, com autoridade intelectual
- Parágrafos concisos (máximo 5 linhas) com uma ideia central por parágrafo
- Frases de transição entre seções para manter o fluxo narrativo
- Evite clichês ("nos dias de hoje", "é fundamental que", "não é segredo que")

### Formatação Rica (OBRIGATÓRIO)
- **Títulos e subtítulos**: use hierarquia clara (H2 para seções, H3 para subseções)
- **Negrito**: para termos-chave, conceitos e definições na primeira ocorrência
- **Itálico**: para ênfase contextual e títulos de publicações
- **Listas ordenadas**: para processos, etapas e procedimentos sequenciais
- **Listas com marcadores**: para enumerações não sequenciais e características
- **Tabelas**: ao menos uma por módulo para comparações, frameworks ou sínteses
- **Blocos de citação (>)**: para insights centrais e conceitos memoráveis
- **Blocos de código**: para exemplos técnicos, comandos e configurações

### Ortografia e Acentuação (INVIOLÁVEL)
- Português do Brasil com acentuação completa e ortografia correta segundo o Acordo Ortográfico vigente
- Palavras com acento obrigatório: não, você, também, até, já, só, após, então, produção, informação, educação, solução, aplicação, função, módulo, conteúdo, tópico, prática, técnica, básico, lógica, página, código, método, número, único, válido, análise, possível, disponível, útil, fácil, difícil, necessário, obrigatório, será, está, início, índice, conclusão, seção, lição, atenção, compreensão, específico, diagnóstico, estratégico, didático, pedagógico
- NUNCA escrever sem acento: "nao", "voce", "producao", "conteudo", "modulo", "analise", "pratica", "tecnica"
- NUNCA adicionar acentos em URLs, slugs, nomes de variáveis ou código-fonte

### Profundidade de Conteúdo
- Cada módulo deve ter entre 2.500 e 4.000 palavras de conteúdo principal
- Priorize profundidade sobre abrangência — é melhor cobrir 3 conceitos bem do que 10 superficialmente
- Inclua dados quantitativos sempre que disponíveis (porcentagens, valores, métricas)
- Cite fontes quando usar dados ou pesquisas específicas
